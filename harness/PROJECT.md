# PROJECT.md — 바이브코딩 보안센터

> **불변 문서(immutable).** 여기 쓰인 내용을 바꾸려면 `harness/DECISIONS.md`에 
> ADR로 남기고 사람이 승인해야 합니다.

## 비전 한 문장

> 바이브코딩으로 서비스를 만든 비개발자가, **두려움 없이 자기 서비스의 건강 상태를 
> 점검하고 필요한 처방을 받을 수 있는 동네 수리점**.

## 제품 메타포: 컴퓨터 수리점

| 수리점 | VibeSec |
|-------|---------|
| 손님이 고장난 PC를 들고 온다 | 사용자가 깃허브 레포/URL을 들고 온다 |
| 사장님이 부팅해보고 진단 | 앱이 자동 스캔해서 쉬운 말로 설명 |
| "이건 쿨러 청소 패키지 4만원" | "이건 KEY RESCUE 패키지 89,000원" |
| 수리 완료 후 A/S 30일 | 보안 수정 후 30일 재발 보증 |
| 단골은 월 관리 계약 | 월 구독 모니터링 |

**이 메타포는 앱 전체 UX의 뼈대입니다. 바꿀 수 없습니다.**

## 타겟 페르소나 (상세는 product-strategist가 확장)

### P1. "내 사이드프로젝트가 걱정되는 기획자" (메인)
- 30대 기획자/디자이너
- 주말에 Cursor로 개인 랜딩페이지 또는 간단한 MVP 만듦
- OpenAI API를 붙여서 GPT 기능 하나 넣었는데 요금 폭탄 소문 들음
- **원하는 것**: "안전한지 아닌지만 알려줘. 안 안전하면 알아서 고쳐줘."

### P2. "런칭한 1인 창업자" (긴급 수요)
- 이미 트래픽이 좀 생긴 상태
- 아직 결제는 안 붙였지만 붙이려고 함
- 가입자가 생기면서 "혹시 내 DB가 공개 상태면 어쩌지?" 불안
- **원하는 것**: 풀 감사 + 결제 붙이기 전 보증

### P3. "이미 털린 사람" (911)
- 갑자기 OpenAI 요금이 이상하거나 DB 데이터가 없어졌거나
- 사색이 되어서 검색해서 찾아옴
- **원하는 것**: 지금 당장. 새벽이든 주말이든.

## 핵심 사용자 여정 (3분 안에 북극성)

```
[앱 실행]
   ↓ 0:00
[환영 화면] "프로젝트 하나만 보여주시면 무료로 진단해드려요"
   ↓ 0:15
[입력] GitHub URL 또는 배포 URL 붙여넣기
   ↓ 0:30
[자동 진단 중] (Liquid Glass 로딩, 재밌는 카피)
   ↓ 1:30
[건강 검진표] 
  "🔴 급해요: 2개 / 🟡 조심해요: 5개 / 🟢 괜찮아요: 12개"
  각 항목은 한 문장으로 비유와 함께 설명
   ↓ 2:00
[추천 처방] 
  "이 상태엔 KEY RESCUE + AUTH FIX 패키지가 딱이에요"
  묶음 할인 보여주기
   ↓ 2:30
[결제 or 카톡 상담]
   ↓ 3:00
```

**3분 안에 여기까지 안 오면 UX 실패.** 모든 에이전트는 이걸 기준으로 작업한다.

## 패키지 체계 (content-curator가 확장해서 312개로 만듦)

### 카테고리 8개
1. **SECRETS** — API 키·환경변수·시크릿 노출
2. **AUTH** — 로그인·세션·JWT·OAuth
3. **DATA** — DB·RLS·백업·유출 방지
4. **NETWORK** — CORS·CSP·Rate Limit·DDoS
5. **INJECTION** — SQL·NoSQL·Prompt·XSS
6. **INFRA** — 배포·도메인·HTTPS·의존성
7. **MONITOR** — 로그·알림·침해 감지
8. **RESPONSE** — 사고 대응·복구·포렌식

### 가격 사다리 (고정)
| 가격대 | 역할 | 예시 |
|--------|------|------|
| 무료 | 리드 | BASIC SCAN |
| 3-9만원 | 단일 증상 | KEY RESCUE |
| 10-19만원 | 영역 패키지 | AUTH FIX, RLS LOCKDOWN |
| 30-50만원 | 풀 감사 | FULL AUDIT ⭐ |
| 80만원+ | 긴급·복구 | INCIDENT 911 |
| 월 19만원+ | 구독 | MONTHLY WATCH |

312개 패키지 = 8 카테고리 × 평균 39개 (단일 + 번들 + 티어별 변주).

## 기술적 제약 (불변)

1. **프론트엔드에 OpenAI 같은 외부 API 키 절대 금지** — 우리 앱이 우리 교훈을 어기면 망신
2. **오프라인 우선** — 네트워크 없이도 앱이 실행되고 과거 리포트를 볼 수 있어야 함
3. **데이터 최소 수집** — 사용자가 URL 붙여넣기 전까지는 텔레메트리도 안 보냄
4. **iOS 26 Liquid Glass 디자인 언어** — ios26-design-system 패키지 사용, 커스텀 최소화
5. **한국어 1급** — 모든 카피는 한국어로 먼저 쓰고 영어는 i18n 키로 분리

## 성공의 정의

MVP 수준:
- [ ] 3분 북극성 체험이 실제로 재현된다
- [ ] 비개발자 3명이 테스트했을 때 모두 "뭔지 알겠다"고 답함  
- [ ] 312개 패키지 카탈로그가 완성되어 있음
- [ ] 레이아웃 깨짐 0건 (pretext 검증 통과)
- [ ] Lighthouse 접근성 ≥ 90

## 비목표 (명시적으로 안 하는 것)

- 실제 보안 스캔 엔진 구현 (이 하네스는 **앱**을 만드는 거지 스캐너를 만드는 게 아님.

- 다국어 실제 번역 (i18n 구조만 만들고 한국어만 채움)
