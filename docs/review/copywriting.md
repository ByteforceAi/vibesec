## 2. 항목별 상세 분석

### 2-1. 헤드라인 효과

**현재**: "바이브코딩 보안. 그게 저희 일입니다."

강점:
- 카테고리를 정의하는 선언형 헤드라인. "바이브코딩 보안"이라는 신조어 카테고리를 만들고 소유권을 주장함.
- 건조하고 자신감 있는 톤이 브랜드 정체성과 맞음.

약점:
- 고객의 **고통(pain)**이 빠져 있음. 방문자는 자기 문제에 대한 이야기를 듣고 싶은데, 이 헤드라인은 회사 소개에 가까움.
- "바이브코딩"이라는 단어를 모르는 방문자에게는 의미가 전달되지 않음. 검색이나 광고로 유입된 사람이 첫 3초 안에 "여기가 내 문제를 해결해주는 곳이구나"를 느끼기 어려움.
- 서브카피 "프로젝트 하나만 보여주시면 무료로 건강검진 해드려요"가 훨씬 강력한데, 계층 구조상 뒤로 밀려 있음.

### 2-2. CTA 텍스트

| 위치 | 현재 CTA | 평가 |
|------|---------|------|
| 홈 히어로 | "레포 점검 시작" | "레포"는 비개발자에게 낯선 단어. 행동은 명확하지만 보상이 없음 |
| 홈 빈 상태 | "첫 진단 받기" | 좋음. "첫"이 진입장벽을 낮춤 |
| 요금제 각 플랜 | "점검 예약" / "정밀 점검 예약" / "상담 후 진행" | 일관적이나, 모든 CTA가 /incident 로 이동하므로 차별화 부족 |
| 요금제 푸터 | "상담 예약" | 반복적. "뭐가 필요한지 모르겠으면 일단 오세요"라는 카피가 강력한데 CTA가 그 에너지를 이어받지 못함 |
| 진단 | "점검 시작" | 기능적이지만 무미건조 |
| 긴급 | "카카오톡 상담" / "전화 상담" | 명확. 다만 긴급성을 고려하면 더 강하게 가도 됨 |

### 2-3. 가격 프레이밍

현재 구조: 기본 39만 / 정밀 89만(추천) / 풀 190만

강점:
- 3단 구조가 명확하고, "추천" 태그가 정밀 점검으로 시선을 유도함.
- "소요 2~3시간", "수정 가이드 PDF 포함" 같은 산출물 명시가 좋음.
- 터미널 데모(`.env` 노출)가 "왜 필요한지"를 직관적으로 보여줌.

약점:
- **비교 앵커가 없음**. 39만원이 비싼지 싼지 판단할 기준이 없음. "프리랜서 개발자 시급 대비", "AWS 요금 폭탄 사례", "데이터 유출 평균 피해액" 같은 비교군이 필요.
- **ROI 프레이밍 부재**. "39만원으로 수백만원 사고를 예방합니다" 같은 투자 관점이 없음.
- 풀 정비 190만원의 가치 설명이 "엔지니어가 직접 고칩니다. 배포까지." 한 줄로 부족. 이 가격대의 고객은 더 많은 확신이 필요함.

### 2-4. FAQ 품질

전반적으로 우수함. 특히:
- "바이브코딩이 뭔가요?" -- 타겟 고객이 실제로 물을 법한 질문이고, 답변이 카테고리 교육 역할을 함.
- "개발을 못 해도 쓸 수 있나요?" -- 핵심 불안을 직접 해소.
- "문제가 없으면요?" -- 구매 저항을 줄이는 좋은 질문. "점검한 레포 대부분에서 3~7개 이슈가 나옵니다"라는 통계가 설득력 있음.

부족한 질문:
- "다른 보안 점검 서비스와 뭐가 다른가요?" -- 경쟁 차별화 질문이 없음.
- "점검 결과를 받으면 제가 직접 고칠 수 있나요?" -- 비개발자의 핵심 불안.
- "우리 서비스가 이미 해킹당한 건 아닌지 어떻게 아나요?" -- 현재 상태에 대한 불안.

### 2-5. 톤 일관성

"동네 수리점 사장님" 톤이 대부분 잘 유지됨:
- 홈: "프로젝트 하나만 보여주시면 무료로 건강검진 해드려요" -- 수리점 톤 완벽.
- 온보딩: "저희가 알아서 살펴볼게요" -- 친근하고 안심시키는 톤.
- 진단 프로그레스: "비밀 보관함을 살펴보고 있어요", "울타리에 구멍이 없는지 보고 있어요" -- 비유가 매우 좋음.
- 요금제 푸터: "뭐가 필요한지 모르겠으면 일단 오세요. 보고 말씀드리겠습니다." -- 이 카피가 사이트 전체에서 가장 강력.

톤이 깨지는 곳:
- FAQ 내 "Pro와 Full Audit 차이" -- "Full Audit"이라는 영문 용어가 갑자기 등장. 페이지에서는 "풀 정비"인데 FAQ에서만 "Full Audit".
- 기능 비교표의 "RLS", "API 엔드포인트 검증", "SLA" -- 비개발자에게 의미 불명. 괄호 설명 없이 단독 사용됨.
- 헤더의 "Byteforce Security" -- 전체 톤이 한국어 친근함인데, 영문 브랜드명이 차갑게 느껴짐.

### 2-6. 공포 vs 안심

전반적으로 잘 균형 잡힘:
- "해킹당합니다!" 류의 공포 표현 전혀 없음.
- 터미널 데모가 "이런 상태일 수 있어요"를 보여주는 방식이 사실적이면서도 위협적이지 않음.
- "312개 레포 점검 기준. 62%에서 유사 이슈 발견" -- 통계로 긴급성을 만들되 공포는 아님.
- 긴급 페이지의 "괜찮아요, 저희가 도와드릴게요" -- 이미 문제가 발생한 고객에게 안심을 먼저 줌.

개선 여지:
- 무료 스캔 FAQ에서 "하룻밤 사이에 요금 폭탄이 나올 수 있는 것들입니다" -- 톤 대비 살짝 공포 쪽으로 기울어짐. "방치하면 불필요한 비용이 생길 수 있어요" 정도가 더 적절.

### 2-7. 구체성

좋은 구체성:
- 312개 레포, 62% 이슈 발견율
- 소요 시간: 2~3시간, 1~2일, 3~7일
- 모니터링: 7일, 30일
- SLA: 4시간 이내
- 재점검 보증: 14일, 30일

부족한 구체성:
- "무료 건강검진"이 구체적으로 몇 개 항목을 검사하는지 불명
- "수정 가이드 PDF"가 몇 페이지인지, 어떤 형식인지 불명
- "엔지니어가 직접 고칩니다"의 엔지니어가 누구인지 (경력, 배경) 불명
- 마곡 방문 시 주소나 지도 링크 없음

## 3. 개선 권고 5가지 (우선순위 순)

### 권고 1: 헤드라인을 고객 중심으로 재구성

**문제**: 현재 헤드라인이 회사 선언문이지, 고객의 문제를 말해주지 않음.

**Before**:
```
바이브코딩 보안. 그게 저희 일입니다.
(서브) 프로젝트 하나만 보여주시면 무료로 건강검진 해드려요.
```

**After (안 A -- 문제 인식형)**:
```
AI로 만든 서비스, 보안은 괜찮을까요?
(서브) 프로젝트 주소 하나면 충분해요. 무료로 건강검진 해드립니다.
(바지 텍스트) 바이브코딩 보안 전문 -- Byteforce Security
```

**After (안 B -- 현재 톤 유지하되 보강)**:
```
바이브코딩 보안. 그게 저희 일입니다.
(서브) Cursor, Lovable, v0로 만든 서비스? 주소만 알려주세요. 무료로 어디가 위험한지 찾아드려요.
```

안 B는 기존 브랜드 정체성을 유지하면서, 서브카피에 구체적 도구명을 넣어 타겟 고객이 "나 얘기다"라고 느끼게 만듦.

### 권고 2: 기능 비교표의 기술 용어를 비유로 교체

**문제**: "RLS", "API 엔드포인트 검증", "SLA" 같은 용어가 비개발자에게 의미 없음. CLAUDE.md 가이드라인 위반.

**Before**:
```
인증 체계 점검
데이터베이스 보안 (RLS)
API 엔드포인트 검증
의존성 취약점 분석
긴급 대응 SLA
```

**After**:
```
로그인 보안 점검
데이터 보관함 잠금 상태
뒷문(API) 열려있는지 확인
사용 중인 부품에 불량이 없는지
긴급 출동 보증 시간
```

괄호 병기가 필요하다면: "뒷문 열려있는지 확인 (API 보안)" 형태로.

### 권고 3: 가격에 비교 앵커와 ROI 프레이밍 추가

**문제**: 39만원의 가치를 판단할 기준이 없음.

**Before**:
```
기본 점검
1:1 대면. 레포 전체 보안 스캔 + 리포트.
W 390,000 / 1회
소요 2~3시간. 마곡 방문 또는 화상.
```

**After**:
```
기본 점검
엔지니어가 직접 보고 짚어드립니다. 스캔 리포트까지.
W 390,000 / 1회
소요 2~3시간. 마곡 방문 또는 화상.
(주석) 보안 프리랜서 2시간 컨설팅 평균 50~80만원. 리포트까지 포함된 가격입니다.
```

풀 정비 190만원의 경우:

**Before**:
```
엔지니어가 직접 고칩니다. 배포까지.
```

**After**:
```
엔지니어가 직접 찾고, 고치고, 배포하고, 30일 지켜봅니다.
한 달간 잠 편히 주무셔도 됩니다.
```

### 권고 4: CTA에서 "레포"를 제거하고 보상을 넣기

**문제**: "레포 점검 시작"의 "레포"는 비개발자에게 낯선 용어. CTA에 보상(무료, 3분)이 빠져 있음.

**Before**:
```
(홈 히어로) 레포 점검 시작
(요금제 푸터) 상담 예약
(진단) 점검 시작
```

**After**:
```
(홈 히어로) 무료 건강검진 받기
(요금제 푸터) 무료 상담받기 -- 부담 없어요
(진단) 3분 무료 점검 시작
```

"무료"라는 단어가 반복되는 것이 의도적. 비개발자의 최대 저항은 "이걸 누르면 돈이 나가는 거 아닌가?"임.

### 권고 5: FAQ에 전환을 돕는 질문 2개 추가

**문제**: 현재 FAQ가 정보 제공은 잘 하지만, 구매 결정을 돕는 질문이 부족.

**추가 권고 질문 A**:
```
Q: 직접 고칠 수 있으면 굳이 돈 내야 하나요?
A: 무료 진단만으로도 어디가 위험한지는 알 수 있어요.
   고치는 건 직접 하셔도 되고, 저희한테 맡기셔도 돼요.
   다만 점검한 프로젝트 62%에서 본인이 몰랐던 문제가 나왔어요.
   아는 것만으로도 가치가 있습니다.
```

**추가 권고 질문 B**:
```
Q: 다른 보안 서비스도 있던데, 여기가 다른 게 뭔가요?
A: 대부분의 보안 서비스는 개발자를 위해 만들어져 있어요.
   리포트에 CVE 번호가 나오고, 고치려면 코드를 알아야 하죠.
   저희는 비개발자가 읽을 수 있는 리포트를 드리고,
   고치는 것까지 대신해드릴 수 있어요.
```

## 5. 총평

현재 카피는 CLAUDE.md의 "동네 수리점 사장님" 톤을 대부분 잘 살리고 있고, 공포 마케팅 없이 문제를 전달하는 균형이 좋음. 터미널 데모와 통계(312개, 62%)는 설득력 있는 증거로 작동함. 요금제 푸터의 "뭐가 필요한지 모르겠으면 일단 오세요. 보고 말씀드리겠습니다."는 사이트 전체에서 가장 전환율이 높을 것으로 예상되는 문장.

가장 시급한 개선 3가지:
1. 기능 비교표의 기술 용어 제거 (CLAUDE.md 위반)
2. CTA에서 "레포" 제거 + "무료" 명시
3. 가격 옆에 비교 앵커 한 줄 추가

이 세 가지만 바꿔도 비개발자의 이탈률이 눈에 띄게 줄어들 것임.
