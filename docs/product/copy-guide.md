# 카피 가이드 (Copy Guide)

> product-strategist 산출물. 화면에 표시되는 모든 텍스트는 이 문서의 규칙을 따릅니다.
> content-curator, screen-builder, qa-reviewer가 참조합니다.

## 1. 톤: "동네 수리점 사장님"

### 정의

우리의 말투는 **동네 컴퓨터 수리점 사장님**입니다. 이 사장님은:

- 컴퓨터를 20년 고쳤지만 손님한테는 쉬운 말로 설명해요
- "이건 이런 상태예요"라고 사실만 말하지, "큰일 났어요!"라고 겁을 주지 않아요
- 손님이 "그래서 어떻게 해요?"라고 물으면 바로 답을 줘요
- 가격을 투명하게 말하고, 꼭 필요한 것만 권해요
- 수리 끝나면 "30일 안에 다시 오면 무료로 봐드려요"라고 말해요

### 톤 체크리스트

- [ ] 존댓말인가? (반말 절대 금지)
- [ ] 한 문장에 기술용어가 없는가?
- [ ] 읽고 나면 "그래서 뭘 해야 하는지" 알 수 있는가?
- [ ] 공포감이 아니라 안심을 주는가?
- [ ] 친근하되 가볍지 않은가? (이모지 남발 금지)

## 3. 권장 치환 사전 (금지어/기술용어 -> 비개발자 표현)

| # | 기술 표현 | 비개발자 표현 |
|---|----------|-------------|
| 1 | API Key exposed | 비밀번호가 밖에 보이는 상태 |
| 2 | Environment Variable missing | 비밀 보관함이 비어있는 상태 |
| 3 | RLS disabled | 금고 문이 열려있는 상태 |
| 4 | SQL Injection vulnerability | 입력란으로 침입할 수 있는 상태 |
| 5 | XSS vulnerability | 모르는 사람이 화면에 글을 올릴 수 있는 상태 |
| 6 | CORS misconfigured | 아무 곳에서나 접근할 수 있는 상태 |
| 7 | JWT expired / invalid | 출입증이 만료된 상태 |
| 8 | Authentication bypass | 로그인 없이 들어올 수 있는 상태 |
| 9 | Rate limiting disabled | 한 사람이 무한으로 두드릴 수 있는 상태 |
| 10 | HTTPS not enabled | 자물쇠가 안 걸린 상태 |
| 11 | Dependency outdated | 오래된 부품을 쓰고 있는 상태 |
| 12 | Database password default | 금고 비밀번호가 초기값인 상태 |
| 13 | No backup configured | 백업(복사본)이 하나도 없는 상태 |
| 14 | Logging disabled | 기록(CCTV)이 꺼져있는 상태 |
| 15 | Admin panel exposed | 관리자 방문이 잠기지 않은 상태 |
| 16 | Session hijacking risk | 다른 사람이 내 행세를 할 수 있는 상태 |
| 17 | Data not encrypted | 고객 정보가 잠금 없이 보관된 상태 |
| 18 | CSP missing | 외부 스크립트가 마음대로 들어올 수 있는 상태 |
| 19 | Open redirect | 방문자를 엉뚱한 곳으로 보낼 수 있는 상태 |
| 20 | Prompt injection risk | AI에게 이상한 명령을 내릴 수 있는 상태 |
| 21 | DDoS unprotected | 대량 방문 공격에 무방비 상태 |
| 22 | Secret in git history | 과거 기록에 비밀번호가 남아있는 상태 |
| 23 | OAuth misconfigured | 소셜 로그인 설정이 잘못된 상태 |
| 24 | No rate limit on login | 로그인 시도를 무제한 할 수 있는 상태 |
| 25 | Insecure cookie settings | 방문 기록표가 안전하지 않은 상태 |
| 26 | Server information leak | 서버 정보가 외부에 보이는 상태 |
| 27 | File upload unrestricted | 아무 파일이나 올릴 수 있는 상태 |
| 28 | Error message verbose | 에러 메시지에 내부 정보가 나오는 상태 |
| 29 | No email verification | 이메일 확인 없이 가입할 수 있는 상태 |
| 30 | Credential in plaintext | 비밀번호가 암호화 없이 저장된 상태 |

## 4. Before/After 예시 10쌍

### 예시 1: 진단 결과 제목
- **Before**: "SQL Injection Vulnerability Detected"
- **After**: "입력란으로 침입할 수 있는 상태예요"

### 예시 2: 심각도 설명
- **Before**: "Critical: Authentication bypass allows unauthorized access"
- **After**: "급해요: 로그인 없이 아무나 들어올 수 있는 상태예요"

### 예시 3: 패키지 설명
- **Before**: "RLS Policy Configuration and Database Access Control Setup"
- **After**: "금고 잠금 장치를 제대로 걸어드려요. 내 고객 정보는 주인만 볼 수 있게요."

### 예시 4: 알림 메시지
- **Before**: "Exposed API key detected in public repository"
- **After**: "비밀번호가 밖에 보이고 있어요. 지금 안으로 옮겨드릴게요."

### 예시 5: 상태 표시
- **Before**: "CORS: Wildcard (*) - HIGH RISK"
- **After**: "아무 곳에서나 접근할 수 있는 상태예요. 울타리를 쳐야 해요."

### 예시 6: 에러 메시지
- **Before**: "Scan failed: Invalid GitHub token"
- **After**: "연결이 안 됐어요. 링크를 다시 확인해주세요."

### 예시 7: CTA 버튼
- **Before**: "Run Vulnerability Assessment"
- **After**: "무료 진단 받기"

### 예시 8: 가격표
- **Before**: "Penetration Testing Package - Enterprise"
- **After**: "풀 감사 패키지 - 전체 점검 + 수리 + 30일 보증"

### 예시 9: 완료 메시지
- **Before**: "Remediation complete. 3 CVEs patched."
- **After**: "수리 완료! 3가지 문제를 고쳤어요. 30일간 지켜드릴게요."

### 예시 10: 구독 설명
- **Before**: "Continuous Security Monitoring with Real-time Threat Detection"
- **After**: "매달 정기 점검해드려요. 이상한 낌새가 보이면 바로 알려드려요."

## 6. 문장 길이 규칙

### 화면별 글자 수 제한 (한국어 기준)

| 위치 | 최대 글자 수 | 예시 |
|------|------------|------|
| 화면 제목 | 15자 | "무료 건강검진" |
| 카드 제목 | 15자 | "비밀번호 노출" |
| 본문 한 줄 | 20자 | "금고 문이 열려있어요" |
| 버튼 텍스트 | 10자 | "진단 받기" |
| 배지/태그 | 8자 | "급해요" |
| 알림 메시지 | 25자 | "수리 완료! 30일 보증 시작" |
| 패키지 부제 | 20자 | "노출된 키 응급 회수" |
| 설명 문단 | 40자 x 3줄 이내 | (3문장 이내) |

### 규칙

1. 한 문장에 쉼표는 최대 1개
2. 접속사("그리고", "그래서", "하지만")로 시작하지 않는다
3. 수동태보다 능동태 ("고쳐졌습니다" -> "고쳤어요")
4. "~하실 수 있습니다" 대신 "~할 수 있어요"
5. 물음표로 끝나는 문장은 화면당 1개 이하

## 7. 이모지 사용 규칙

### 허용되는 이모지

| 이모지 | 용도 | 사용 위치 |
|--------|------|----------|
| :red_circle: | 급한 증상 표시 | 건강검진표, 패키지 심각도 |
| :yellow_circle: | 조심할 증상 표시 | 건강검진표, 패키지 심각도 |
| :green_circle: | 괜찮은 상태 표시 | 건강검진표, 패키지 심각도 |

- 위 3개 외 이모지는 사용하지 않습니다
- 텍스트 앞에 이모지를 장식용으로 붙이지 않습니다
- 버튼에 이모지를 넣지 않습니다

## 8. 공포 마케팅 금지 원칙

### 절대 하지 않는 것

- "해킹당합니다!" "위험합니다!" "지금 당장 안 하면!" 류의 공포 유발
- 빨간색 경고 아이콘 남발
- "이미 늦었을 수 있다" 류의 협박
- 피해 금액을 과장해서 보여주기

### 대신 하는 것

| 공포 표현 (금지) | 안심 표현 (권장) |
|-----------------|-----------------|
| "해킹당할 수 있습니다!" | "지금 상태에서 더 안전하게 만들 수 있어요" |
| "데이터가 유출될 위험!" | "고객 정보를 더 안전하게 보관할 수 있어요" |
| "지금 당장 조치하세요!" | "급한 것부터 하나씩 고쳐드릴게요" |
| "이미 털렸을 수도!" | "한번 확인해볼게요. 대부분은 괜찮아요" |
| "수백만원 손해!" | "지금 고치면 요금 걱정은 안 하셔도 돼요" |

### 심각도 표현 가이드

- :red_circle: **급해요**: "지금 고치는 게 좋아요" (공포가 아니라 우선순위)
- :yellow_circle: **조심해요**: "시간 될 때 봐두면 좋아요"
- :green_circle: **괜찮아요**: "잘 돼있어요" 또는 "나중에 해도 돼요"

## 9. 수리점 어휘 일관성 체크

앱 전반에서 아래 어휘를 일관되게 사용합니다. 동의어가 섞이면 안 됩니다.

| 개념 | 공식 용어 | 쓰면 안 되는 동의어 |
|------|----------|-------------------|
| 보안 점검 | 진단 | 스캔, 검사, 분석, 테스트 |
| 보안 문제 | 증상 | 취약점, 이슈, 버그, 결함 |
| 수리 서비스 | 처방 | 솔루션, 서비스, 플랜, 패치 |
| 수리 행위 | 수리 | 수정, 패치, 픽스, 리미디에이션 |
| 사후 관리 | A/S | 지원, 서포트, 유지보수 |
| 정기 관리 | 단골 관리 | 구독, 멤버십, 월정액 |
| 긴급 서비스 | 긴급출동 | 응급, 인시던트, 핫라인 |
| 결과 리포트 | 건강검진표 | 리포트, 보고서, 대시보드 |
| 구매 | 접수 | 결제, 구독, 구매 |

## 10. 자가검증 체크리스트

카피를 작성한 후 아래를 확인합니다:

- [ ] 금지어 20개가 본문에 포함되지 않았는가?
- [ ] 모든 문장이 글자 수 제한을 지키는가?
- [ ] 수리점 어휘가 일관되게 사용됐는가?
- [ ] 공포 표현이 아닌 안심 표현을 썼는가?
- [ ] 비유가 자연스럽고 일관되는가?
- [ ] P1(기획자)이 읽었을 때 "아, 이건 나한테 하는 말이다" 느낄 수 있는가?
