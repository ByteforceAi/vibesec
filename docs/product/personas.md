## P1. "내 사이드프로젝트가 걱정되는 기획자" (메인 페르소나)

### 기본 정보

- **이름**: 김지현 (가상)
- **나이**: 33세
- **직업**: IT 회사 서비스 기획자 (5년차)
- **거주지**: 서울 마포구
- **기술 수준**: Cursor로 간단한 웹앱을 만들 수 있음. 터미널에서 `npm run dev`는 치지만, `.env` 파일이 뭔지는 "환경 설정 파일인가?" 정도로 이해

### 오늘 하루 일상

- **07:30** 알람. 침대에서 인스타 피드 훑음. 요즘 "나만의 앱 만들기" 릴스가 자꾸 뜸
- **08:30** 출근길 지하철. 트위터(X)에서 "Cursor로 주말에 SaaS 만들었다" 글 읽고 북마크
- **09:00~18:00** 회사. 서비스 기획서 쓰고 디자이너/개발자와 미팅
- **19:30** 퇴근 후 카페. 노트북 열고 주말에 만든 사이드 프로젝트 이어서 작업
- **21:00** 집. 유튜브에서 "Cursor 튜토리얼" 보다가 댓글에서 "API 키 노출되면 큰일" 이라는 말 발견
- **21:30** 불안해져서 "바이브코딩 보안" 검색. VibeSec 발견

### 바이브코딩으로 뭘 만들었는가

- Cursor + Claude로 개인 포트폴리오 랜딩페이지 제작
- OpenAI API를 붙여서 "내 이력서를 요약해주는 기능" 하나 넣음
- Supabase를 DB로 쓰는데 설정은 유튜브 따라함

### 보안을 모르는 증거 (실제로 할 법한 대사)

- "`.env` 파일? 그거 Cursor가 알아서 만들어준 거 아닌가?"

- "Supabase 세팅할 때 anon key를 프론트에 넣으라길래 넣었는데, 이게 위험한 건가?"
- "CORS? 그거 에러 나길래 스택오버플로우에서 전부 허용하라는 코드 복붙했어"

### 뭘 두려워하는가 (3가지)

1. **요금 폭탄**: OpenAI API 키가 털려서 하룻밤 사이에 수십만원 청구되는 것
2. **창피함**: 친구들한테 보여준 사이트가 해킹당해서 이상한 페이지가 뜨는 것
3. **데이터 유출**: Supabase에 저장한 친구들 이메일이 어디서 돌아다니는 것

### 뭘 들으면 안심하는가 (3가지 멘트)

2. "이건 오늘 안에 고칠 수 있어요. 제가 다 해드릴게요."
3. "지금 고치면 요금 걱정은 안 하셔도 돼요."

- **11:00** 지인 개발자에게 카톡: "혹시 내 사이트 DB가 공개 상태일 수 있어?" 답장: "Supabase RLS 켜놨어?" "RLS가 뭔데?"

### 보안을 모르는 증거 (실제로 할 법한 대사)

- "RLS? 아, 그거 bolt.new가 알아서 해준 거 아냐?"

3. **신뢰 상실**: 가입자들이 "이 서비스 위험하다"고 SNS에 올리는 것

### 뭘 들으면 안심하는가 (3가지 멘트)

2. "고객 데이터는 지금 이 상태면 안전해요. 여기 하나만 더 잠그면 완벽해요."

## 페르소나 공통 특성

### 공통점
- 코딩은 AI가 해줬고, 본인은 "기획"과 "지시"를 한 사람
- 보안이라는 단어는 알지만 구체적으로 뭘 해야 하는지 모름
- 영문 기술용어를 보면 "이건 내가 볼 게 아니구나" 하고 떠남
- 믿을 수 있는 전문가가 "괜찮다"고 말해주길 원함

### 차이점

| 구분 | P1 기획자 | P2 창업자 | P3 털린 사람 |
|------|----------|----------|-------------|
| 긴급도 | 낮음 (불안) | 중간 (곧 결제 도입) | 극도로 높음 (사고 발생) |
| 예산 감수성 | 높음 (무료~5만원) | 중간 (30만원까지 가능) | 낮음 (지금 해결만 되면) |
| 앱 진입 경로 | 유튜브/블로그 검색 | 지인 추천/검색 | "긴급 보안" 검색 |
| 가장 원하는 패키지 | 무료 진단 + 단일 수리 | 풀 감사 | 긴급출동 (911) |
| 구독 전환 가능성 | 낮음 | 높음 | 중간 (사고 후 불안) |

### 페르소나별 핵심 화면 진입 시나리오

- **P1**: 홈 화면 -> 무료 진단 -> 건강검진표 -> "이것만 고치면 돼요" -> 단일 패키지 결제
- **P2**: 홈 화면 -> 무료 진단 -> 건강검진표 -> "결제 도입 전 필수 항목" -> 풀 감사 패키지 결제
- **P3**: 긴급출동 진입 -> 즉시 상담 연결 -> 긴급 패키지 -> 이후 모니터링 구독
