# 사용자 여정 (Journey Map)

> product-strategist 산출물. screen-builder가 이 문서를 설계도로 읽고 라우트를 만듭니다.
> 3분 북극성 체험을 화면 단위로 분해합니다.

---

## 전체 흐름 요약

```
[S0 앱 실행]  0:00
     |
[S1 환영 화면]  0:00~0:15
     |
[S2 URL 입력]  0:15~0:30
     |
[S3 진단 중]  0:30~1:30
     |
[S4 건강검진표]  1:30~2:00
     |
[S5 추천 처방]  2:00~2:30
     |
[S6 접수/상담]  2:30~3:00
```

### 페르소나별 분기

```
P1 기획자:  S0 -> S1 -> S2 -> S3 -> S4 -> S5(단일 패키지) -> S6(결제)
P2 창업자:  S0 -> S1 -> S2 -> S3 -> S4 -> S5(풀 감사) -> S6(결제)
P3 털린 사람: S0 -> S1 -> S7(긴급출동) -> S6(카톡 상담)
```

---

## S0. 앱 실행 (스플래시)

- **라우트**: `/` (초기 로딩)
- **목적**: 앱이 살아있다는 신호. 브랜드 각인
- **체류 시간**: 1~2초 (자동 전환)
- **사용자가 보는 것**:
  - 바이브코딩 보안센터 로고
  - "동네 수리점" 느낌의 따뜻한 색감
  - Liquid Glass 효과가 살짝 적용된 로고 애니메이션
- **사용자가 느끼는 것**: "깔끔하다. 무섭지 않다."
- **사용자가 하는 행동**: 없음 (자동 전환)
- **핵심 카피**: 없음 (로고만)
- **성공 조건**: 2초 안에 S1으로 전환
- **분기**: 
  - 최초 실행: S1(환영 화면)으로
  - 재방문: 바로 홈(S2 변형)으로

---

## S1. 환영 화면 (온보딩)

- **라우트**: `/onboarding`
- **목적**: 앱이 뭘 해주는지 3초 안에 이해시키기. 공포가 아닌 안심의 첫인상
- **체류 시간**: 10~15초
- **사용자가 보는 것**:
  - 큰 제목: "프로젝트 건강검진" (15자 이내)
  - 부제: "링크 하나면 무료로 진단해드려요" (20자 이내)
  - 일러스트: 수리점 사장님이 웃으며 청진기 대는 이미지 (메타포 시각화)
  - CTA 버튼: "무료 진단 받기" (하단 고정)
  - 하단 텍스트: "3분이면 끝나요"
- **사용자가 느끼는 것**: "무료래. 3분이면 끝나래. 한번 해볼까."
- **사용자가 하는 행동**: "무료 진단 받기" 버튼 탭
- **핵심 카피**:
  - 제목: "프로젝트 건강검진"
  - 부제: "링크 하나면 무료로 진단해드려요"
  - CTA: "무료 진단 받기"
  - 보조: "3분이면 끝나요"
- **성공 조건**: 80% 이상이 CTA를 탭
- **이탈 리스크**: "보안"이라는 단어에 "나한테 해당 안 되는 것" 느낌
- **이탈 대응**: "보안"을 "건강검진"으로 프레이밍. 가볍고 무료라는 점 강조

### P3 전용 분기

- 화면 하단에 작은 링크: "지금 급한 상황이에요" 
- 탭하면 S7(긴급출동)으로 직행

## S2. URL 입력 (진단 접수)

- **라우트**: `/diagnose`
- **목적**: 사용자의 프로젝트를 받아서 진단 시작
- **체류 시간**: 10~15초
- **사용자가 보는 것**:
  - 큰 입력란 하나 (가운데 배치)
  - 플레이스홀더: "깃허브 주소나 사이트 주소를 붙여넣으세요"
  - 입력란 아래 예시: "예: github.com/내아이디/내프로젝트"
  - "진단 시작" 버튼 (입력 후 활성화)
  - 하단 안심 문구: "코드를 저장하거나 외부에 보내지 않아요"
- **사용자가 느끼는 것**: "간단하네. 주소만 넣으면 되는 거야?"
- **사용자가 하는 행동**: URL 붙여넣기 -> "진단 시작" 탭
- **핵심 카피**:
  - 제목: "어디 한번 볼게요"
  - 플레이스홀더: "주소를 붙여넣으세요"
  - CTA: "진단 시작"
  - 안심 문구: "코드를 저장하지 않아요"
- **성공 조건**: URL 입력 후 10초 내 "진단 시작" 탭
- **이탈 리스크**: 
  1. "깃허브 주소가 뭐지?" (URL 형식을 모름)
  2. "내 코드를 보는 거야? 무섭다"
- **이탈 대응**:
  1. 입력란 아래 "어떤 주소를 넣어야 하나요?" 도움말 링크
  2. "코드를 저장하거나 외부에 보내지 않아요" 안심 문구 상시 노출

### 빈 상태

- URL을 안 넣고 버튼을 누르면: "주소를 먼저 넣어주세요" (부드러운 안내)

### 오류 상태

- 잘못된 URL: "이 주소는 확인할 수 없어요. 다시 확인해주세요"
- 비공개 저장소: "이 프로젝트는 비공개라 접근이 안 돼요. 공개 저장소이거나 배포된 사이트 주소를 넣어주세요"
- 네트워크 오류: "인터넷 연결을 확인해주세요"

## S3. 진단 중 (로딩)

- **라우트**: `/diagnose` (같은 라우트, 상태 변환)
- **목적**: 기다리는 시간을 지루하지 않게. 신뢰감 형성
- **체류 시간**: 30~60초 (실제 스캔 시간 또는 목업 타이머)
- **사용자가 보는 것**:
  - Liquid Glass 스타일 진행 바 (프로그레스)
  - 단계별 메시지가 바뀜:
    1. "비밀 보관소를 살펴보고 있어요..." (0~20%)
    2. "출입문을 확인하고 있어요..." (20~40%)
    3. "금고 상태를 점검 중이에요..." (40~60%)
    4. "울타리를 둘러보고 있어요..." (60~80%)
    5. "마지막으로 건물 상태를 봐요..." (80~100%)
  - 하단에 재미 요소: "수리점 사장님이 꼼꼼히 보는 중..."
- **사용자가 느끼는 것**: "뭔가 진짜 확인하고 있구나. 꼼꼼하네."
- **사용자가 하는 행동**: 기다림 (다른 화면 전환 없이)
- **핵심 카피**: 단계별 메시지 (위 참조)
- **성공 조건**: 60초 이내 완료. 사용자가 중간에 나가지 않음
- **이탈 리스크**: "너무 오래 걸린다"
- **이탈 대응**: 단계별 메시지로 진행 상황을 계속 알려줌. 예상 시간 표시

### 오프라인 상태

- "인터넷이 끊겼어요. 연결되면 이어서 진단할게요" + 대기 화면

### 오류 상태

- 진단 실패: "죄송해요, 진단 중 문제가 생겼어요. 다시 한번 해볼까요?" + "다시 진단" 버튼

## S4. 건강검진표 (진단 결과)

- **라우트**: `/report/[scanId]`
- **목적**: 결과를 한눈에 이해시키기. 공포가 아닌 "현재 상태 파악"
- **체류 시간**: 20~30초
- **사용자가 보는 것**:
  - 상단 요약 카드:
    - "급해요 :red_circle: 2개"
    - "조심해요 :yellow_circle: 5개"
    - "괜찮아요 :green_circle: 12개"
  - 종합 점수 또는 상태: "지금 상태: 좀 손볼 데가 있어요" (공포 아닌 중립)
  - 항목 리스트 (심각도순 정렬):
    - 각 항목: 아이콘 + 한 줄 설명 + 비유
    - 예: :red_circle: "비밀번호가 밖에 보이고 있어요 (가게 비밀번호를 문 앞에 붙여놓은 상태)"
    - 예: :green_circle: "자물쇠가 잘 걸려있어요"
  - 하단 CTA: "추천 처방 보기"
- **사용자가 느끼는 것**: "아, 2개만 급하고 나머진 괜찮구나. 다행이다."
- **사용자가 하는 행동**: 
  1. 항목 스크롤해서 읽기
  2. "추천 처방 보기" 탭
- **핵심 카피**:
  - 제목: "검진 결과가 나왔어요"
  - 요약: "급한 건 N개, 나머진 괜찮아요"
  - CTA: "추천 처방 보기"
- **성공 조건**: 사용자가 "이해했다"고 느끼고 CTA를 탭
- **이탈 리스크**: 
  1. "빨간 게 있네. 무섭다." -> 이탈
  2. "전부 초록이네. 필요 없네." -> 이탈
- **이탈 대응**:
  1. 빨간 항목에도 "고치면 돼요" 뉘앙스. 공포 금지
  2. 전부 초록이면: "잘 관리하고 계시네요! 정기 점검 받으시면 안심이에요" -> 구독 유도

### P1 분기

- 빨간 항목 1~2개 + "이것만 고치면 돼요" 메시지 강조
- 단일 패키지 추천 (3~9만원대)

### P2 분기

- 빨간 + 노란 항목 다수 + "결제 전에 한번 정리하는 게 좋겠어요" 메시지
- 풀 감사 패키지 추천 (30~50만원대)

### 빈 상태

- 진단할 수 없었을 때: "이 프로젝트는 진단할 수 없었어요" + 재진단 안내

## S5. 추천 처방 (패키지 추천)

- **라우트**: `/report/[scanId]` (하단 섹션 또는 시트)
- **목적**: 진단 결과에 딱 맞는 패키지를 제안. 과잉 판매하지 않기
- **체류 시간**: 20~30초
- **사용자가 보는 것**:
  - "이 상태엔 이 처방이 맞아요" 제목
  - 추천 패키지 카드 1~3개:
    - 패키지 이름 (한국어)
    - 한 줄 설명
    - 가격
    - "이걸 하면 이렇게 돼요" 기대 효과
    - "접수하기" 버튼
  - 묶음 할인 표시 (2개 이상일 때): "같이 하면 N% 할인"
  - "전체 패키지 둘러보기" 링크 (카탈로그로 이동)
- **사용자가 느끼는 것**: "아, 이 두 개만 하면 되는 거구나. 생각보다 비싸지 않네."
- **사용자가 하는 행동**: 
  1. 패키지 카드 읽기
  2. "접수하기" 탭 또는 "전체 패키지 둘러보기"
- **핵심 카피**:
  - 제목: "이 상태엔 이게 맞아요"
  - 패키지 카드: 이름 + 한 줄 설명 + 가격
  - CTA: "접수하기"
  - 할인: "같이 하면 할인돼요"
- **성공 조건**: 50% 이상이 패키지 하나 이상 선택

## S6. 접수/상담 (결제)

- **라우트**: `/checkout`
- **목적**: 결제 또는 상담 연결. 최대한 간단하게
- **체류 시간**: 30~60초
- **사용자가 보는 것**:
  - 선택한 패키지 요약
  - 총 금액
  - 결제 버튼 (MVP에서는 모킹)
  - "카톡으로 상담하기" 대안 버튼
  - "작업 완료 후 알림을 보내드려요"
  - 하단: "30일 A/S 보증" 안내
- **사용자가 느끼는 것**: "간단하네. 카톡으로 물어볼 수도 있고."
- **사용자가 하는 행동**: 결제 또는 카톡 상담 탭
- **핵심 카피**:
  - 제목: "접수하기"
  - 요약: 패키지명 + 금액
  - CTA 1: "결제하기" 
  - CTA 2: "카톡으로 상담하기"
  - 안심: "30일 A/S 보증"
- **성공 조건**: 결제 또는 상담 연결 완료

### P3 전용

- "긴급출동" 패키지 선택 시 카톡 상담이 1순위 CTA
- "지금 바로 연결할게요" 메시지

---

## S7. 긴급출동 (P3 전용 진입점)

- **라우트**: `/incident`
- **목적**: P3("이미 털린 사람")가 최단 경로로 도움을 받게 하기
- **체류 시간**: 15~20초
- **사용자가 보는 것**:
  - 큰 제목: "괜찮아요, 도와드릴게요"
  - 증상 선택 (탭으로 빠르게):
    - "요금이 이상해요"
    - "데이터가 사라졌어요"
    - "사이트가 이상해요"
    - "계정이 털린 것 같아요"
    - "기타"
  - 선택 후 바로: "카톡으로 연결" CTA
  - 응급 안내: "카드사에 먼저 전화해서 결제를 막아두세요" (해당되는 경우)
- **사용자가 느끼는 것**: "다행이다, 여기서 바로 도움받을 수 있겠다."
- **사용자가 하는 행동**: 증상 선택 -> 카톡 연결
- **핵심 카피**:
  - 제목: "괜찮아요, 도와드릴게요"
  - 부제: "증상을 골라주세요"
  - CTA: "지금 바로 상담하기"
  - 응급 안내: "카드사에 전화해서 결제를 막아두세요"
- **성공 조건**: 증상 선택 후 30초 내 상담 연결

## 보조 화면

### 패키지 카탈로그

- **라우트**: `/packages`
- **목적**: 312개 패키지를 탐색할 수 있는 카탈로그
- **진입점**: S5의 "전체 패키지 둘러보기", 또는 탭바
- **사용자가 보는 것**:
  - 8개 카테고리 탭 (비개발자 이름): 비밀 보관소, 출입 관리, 금고 관리, 울타리 관리, 침입 방지, 건물 관리, 감시 카메라, 긴급출동
  - 각 카테고리 내 패키지 카드 리스트
  - 가격 필터 (무료 / 10만원 미만 / 10~30만원 / 30만원 이상 / 월 구독)
  - 심각도 필터 (:red_circle: / :yellow_circle: / :green_circle:)

### 패키지 상세

- **라우트**: `/packages/[id]`
- **사용자가 보는 것**:
  - 패키지 이름 + 한 줄 설명
  - "이런 증상일 때 필요해요" (증상 목록)
  - "이렇게 수리해드려요" (작업 내용)
  - "수리 후 이렇게 달라져요" (기대 효과)
  - 가격 + "접수하기" 버튼
  - "30일 A/S 보증" 배지

## 엣지케이스 분기

### 오프라인 상태
- S0~S1: 정상 작동 (온보딩은 오프라인 가능)
- S2: "진단하려면 인터넷이 필요해요" 안내
- S4: 과거 검진표는 오프라인에서도 조회 가능

- 서버: "잠시 후 다시 시도해주세요. 계속되면 카톡으로 알려주세요."
