# Byteforce 전체 리뉴얼 계획

## 현재 문제 요약
1. 하단 네비가 "홈|진단|리포트|요금제" — 자동 스캔 SaaS 구조의 잔재
2. diagnose 페이지가 fake-scanner 자동 스캔 — 비즈니스 모델과 충돌
3. report 페이지가 가짜 스캔 결과 — 의미 없음
4. checkout 페이지가 온라인 결제 mock — 불필요
5. onboarding이 자동 스캔 전제 — 잘못된 기대
6. 입금 후 예약 확정 플로우 없음
7. "세션 로그 보기" 등 잔여 개발 텍스트
8. packages/[id] 312개 개별 페이지 — 직접 노출 불필요

## 새 구조 (10개 → 5개)

### 유지 + 개선
1. `/` (홈) — OS 쉘, 코어 오브제, 서비스 카드. CTA를 상담예약으로.
2. `/packages` (요금제) — 4티어 + Enterprise. 지점 정보. FAQ.
3. `/contact` (상담 예약) — 현재 incident 챗봇 유지. "입금 후 예약 확정" 안내 추가.

### 새로 만들기
4. `/check` (자가 진단) — 5문항 체크리스트. 결과가 항상 상담으로 수렴.
5. `/sample` (보고서 샘플) — "이런 보고서를 받게 됩니다" 미리보기.

### 제거
- `/diagnose` → `/check`로 대체
- `/report` → `/sample`로 대체
- `/report/[scanId]` → 제거
- `/checkout` → 제거 (입금은 계좌이체)
- `/onboarding` → 제거
- `/packages/[id]` → 라우트는 남기되 사이트에서 링크 제거

### 네비게이션 변경
```
현재: 홈 | 진단 | 리포트 | 요금제
변경: 홈 | 자가진단 | 요금제 | 상담예약
```

## 입금 후 예약 확정 플로우
챗봇 상담 완료 후:
1. "접수되었습니다. 24시간 내 이메일로 견적서를 보내드립니다."
2. 견적서에 계좌 정보 포함
3. "입금 확인 후 점검 일정이 확정됩니다."
4. 입금 확인 → 일정 확정 이메일 발송

사이트에서는 챗봇 완료 시:
"견적서를 이메일로 보내드립니다. 입금 확인 후 점검 일정이 확정됩니다."

## 실행 순서
1. 하단 네비 변경 (모든 페이지)
2. `/check` 자가진단 페이지 생성
3. `/sample` 보고서 샘플 페이지 생성
4. 홈 CTA → /contact로 변경
5. 챗봇에 "입금 후 확정" 안내 추가
6. 불필요 페이지 제거/리다이렉트
7. 빌드 + 배포
