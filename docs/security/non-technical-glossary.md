# 비기술 용어 사전 (Non-Technical Glossary)

> 기술 용어를 비개발자가 이해할 수 있는 일상어로 번역한 사전입니다.
> screen-builder가 툴팁/도움말을 만들 때, content-curator가 카피를 쓸 때 1차 참조 자료입니다.

---

## 사용법

- **기술용어**: 개발자/보안 전문가가 쓰는 원래 용어
- **일상어 표현**: 비개발자에게 보여줄 때 쓰는 표현
- **비유**: 일상 경험에 빗대어 설명할 때 쓰는 비유
- **한줄 설명**: 비개발자에게 한 문장으로 설명할 때

---

## 전체 목록

| # | 기술용어 | 일상어 표현 | 비유 | 한줄 설명 |
|---|---------|-----------|------|----------|
| 1 | API Key | 서비스 비밀번호 | 가게 금고 비밀번호 | AI나 외부 서비스를 쓸 수 있는 비밀번호예요 |
| 2 | Secret Key | 관리자 비밀번호 | 마스터키 | 모든 권한을 가진 최고 비밀번호예요 |
| 3 | Environment Variable (.env) | 비밀번호 보관 파일 | 금고 안 서랍 | 비밀번호들을 모아둔 안전한 파일이에요 |
| 4 | Git / GitHub | 코드 보관소 | 서류 보관함 | 코드의 모든 버전을 저장하는 곳이에요 |
| 5 | Git History | 코드 변경 기록 | 서류 수정 이력 | 과거에 코드를 어떻게 바꿨는지 기록이에요 |
| 6 | Frontend | 화면(눈에 보이는 부분) | 가게 매장 | 손님이 직접 보고 만지는 부분이에요 |
| 7 | Backend / Server | 서버(뒤에서 일하는 부분) | 가게 주방/창고 | 손님 눈에 안 보이는 곳에서 일하는 부분이에요 |
| 8 | Database (DB) | 데이터 저장소 | 금고 | 고객 정보, 주문 기록 등을 보관하는 곳이에요 |
| 9 | RLS (Row Level Security) | 데이터 열람 권한 | 금고 칸막이 잠금장치 | 사람마다 자기 칸만 볼 수 있게 하는 장치예요 |
| 10 | SQL | 데이터 조회 언어 | 금고에 말 거는 방법 | 데이터를 찾고, 넣고, 바꿀 때 쓰는 명령어예요 |
| 11 | SQL Injection | 입력란을 통한 데이터 침입 | 주문서에 "금고 열어줘" 끼워넣기 | 입력란에 특수 명령을 넣어 데이터를 빼가는 거예요 |
| 12 | NoSQL Injection | 검색을 통한 데이터 침입 | 도서관 검색대에서 사서에게 몰래 지시하기 | SQL 인젝션과 같은 원리이지만 다른 종류의 데이터베이스를 노려요 |
| 13 | XSS (Cross-Site Scripting) | 화면에 악성 코드 심기 | 게시판에 요술 포스터 붙이기 | 댓글 같은 곳에 코드를 숨겨서 보는 사람을 공격해요 |
| 14 | Prompt Injection | AI 꼬드기기 공격 | 가게 직원한테 "사장님이 공짜로 주래요" 거짓말하기 | AI에게 원래 규칙을 무시하라고 속여서 이상한 행동을 시켜요 |
| 15 | CORS | 외부 사이트 접근 규칙 | 가게 뒷문 출입 규정 | 어떤 사이트에서 우리 서버에 요청을 보낼 수 있는지 정하는 규칙이에요 |
| 16 | CSP (Content Security Policy) | 외부 코드 차단 규칙 | 가게 전단지 게시 규정 | 어떤 외부 코드가 우리 사이트에서 실행될 수 있는지 정해요 |
| 17 | Rate Limit | 요청 횟수 제한 | 1인당 주문 수량 제한 | 한 사람이 너무 많은 요청을 보내지 못하게 막는 거예요 |
| 18 | DDoS | 대량 접속 공격 | 가게 앞에 수천 명 동시 몰리기 | 서버가 처리 못 할 정도로 요청을 한꺼번에 보내서 서비스를 멈추게 해요 |
| 19 | JWT (JSON Web Token) | 로그인 확인 딱지 | 극장 입장권 | 로그인했다는 걸 증명하는 디지털 딱지예요 |
| 20 | Session | 로그인 상태 유지 | 출입증 유효 기간 | 한번 로그인하면 일정 시간 동안 로그인이 유지되는 거예요 |
| 21 | OAuth | 소셜 로그인 | 다른 건물 출입증으로 들어오기 | 구글, 카카오 같은 다른 서비스 계정으로 로그인하는 거예요 |
| 22 | HTTPS / SSL | 보안 연결 (자물쇠) | 손님과 가게 사이 유리벽 | 데이터가 오갈 때 중간에서 엿보지 못하게 암호화해요 |
| 23 | Authentication (인증) | 로그인 / 본인 확인 | 출입문 잠금장치 | "당신이 누구인지" 확인하는 절차예요 |
| 24 | Authorization (인가) | 권한 확인 | VIP 라운지 출입 자격 | "이 기능을 쓸 자격이 있는지" 확인하는 절차예요 |
| 25 | Encryption (암호화) | 데이터 잠금 | 서류를 금고에 넣고 잠그기 | 데이터를 읽을 수 없는 형태로 바꿔서 보관하는 거예요 |
| 26 | Hash (해싱) | 비밀번호 변환 저장 | 원본을 녹여서 만든 도장 | 비밀번호를 되돌릴 수 없는 형태로 바꿔서 저장해요 |
| 27 | Middleware | 중간 검사 장치 | 출입구 보안 검색대 | 요청이 서버에 도착하기 전에 검사하는 장치예요 |
| 28 | Webhook | 외부 알림 수신 | 택배 도착 알림 | 다른 서비스에서 우리 서버로 보내는 자동 알림이에요 |
| 29 | Cookie | 브라우저 기록표 | 방문 도장 | 브라우저가 기억하는 작은 정보 조각이에요 |
| 30 | HttpOnly Cookie | 안전한 기록표 | 위조 방지 도장 | 프로그램으로 읽을 수 없게 보호된 기록표예요 |
| 31 | Token | 인증 표시 | 놀이공원 팔찌 | 로그인했음을 증명하는 디지털 표시예요 |
| 32 | Payload | 전송 데이터 | 택배 내용물 | 요청이나 응답에 담긴 실제 데이터예요 |
| 33 | Endpoint | 서버 주소 | 가게 각 창구 | 서버에서 특정 기능을 담당하는 주소예요 |
| 34 | Vulnerability | 약한 곳 / 틈새 | 벽에 난 구멍 | 보안에 문제가 될 수 있는 약한 부분이에요 |
| 35 | Exploit | 약한 곳 이용하기 | 구멍으로 들어가기 | 발견된 약한 곳을 실제로 이용해서 침입하는 거예요 |
| 36 | Penetration Testing | 보안 모의 침입 | 가게 방범 테스트 | 일부러 침입을 시도해서 약한 곳을 찾는 검사예요 |
| 37 | Firewall | 방화벽 | 건물 담장 + 경비원 | 허용되지 않은 접근을 막는 보안 장치예요 |
| 38 | Logging | 기록 남기기 | CCTV 녹화 | 누가 언제 무엇을 했는지 기록하는 거예요 |
| 39 | Monitoring | 실시간 감시 | CCTV 실시간 확인 | 서비스 상태를 계속 지켜보는 거예요 |
| 40 | Alert | 경보 / 알림 | 화재 경보기 | 이상한 일이 생기면 자동으로 알려주는 거예요 |
| 41 | Backup | 백업 (복사본) | 중요 서류 복사 | 데이터를 미리 복사해두어서 사고 시 복구할 수 있게 해요 |
| 42 | Incident Response | 사고 대응 | 119 출동 | 보안 사고가 났을 때 대응하는 절차예요 |
| 43 | Deployment | 배포 | 가게 오픈 | 만든 서비스를 인터넷에 공개하는 거예요 |
| 44 | Dependency | 외부 부품 | 가게 인테리어 자재 | 다른 사람이 만든 코드를 가져다 쓰는 거예요 |
| 45 | Source Map | 코드 설계도 | 건물 도면 | 코드의 원본 구조를 보여주는 파일이에요 |
| 46 | Directory Listing | 파일 목록 공개 | 창고 재고 목록 전시 | 서버의 파일 목록이 외부에서 보이는 상태예요 |
| 47 | Domain | 인터넷 주소 | 가게 간판/주소 | 사람들이 사이트를 찾아올 때 쓰는 주소예요 |
| 48 | DNS | 주소 안내 시스템 | 전화번호부 | 인터넷 주소를 실제 서버 위치로 연결해주는 시스템이에요 |
| 49 | Two-Factor Authentication (2FA) | 2단계 인증 | 출입문 + 지문 인식 | 비밀번호 외에 추가로 한 번 더 확인하는 거예요 |
| 50 | Sanitization | 입력값 정화 | 손님 짐 검사 | 사용자가 입력한 내용에서 위험한 부분을 제거하는 거예요 |
| 51 | HSTS | 보안 연결 강제 | "반드시 정문으로만 입장" 규칙 | 항상 보안 연결(HTTPS)만 사용하도록 강제하는 규칙이에요 |
| 52 | WAF (Web Application Firewall) | 웹 전용 방화벽 | 가게 입구 금속탐지기 | 웹사이트로 들어오는 요청 중 위험한 것을 걸러내요 |
| 53 | Brute Force | 비밀번호 무작위 대입 | 금고 번호를 0000부터 9999까지 다 넣어보기 | 가능한 비밀번호를 하나씩 다 시도하는 공격이에요 |
| 54 | Phishing | 사기 사이트 유도 | 가짜 가게 간판 걸기 | 진짜처럼 생긴 가짜 사이트로 유인해서 정보를 빼가는 거예요 |
| 55 | Redirect | 페이지 이동 | 안내판 따라가기 | 한 페이지에서 다른 페이지로 자동으로 이동시키는 거예요 |
| 56 | Compliance | 법규 준수 | 영업허가 규정 지키기 | 개인정보보호법 같은 법률을 지키는 거예요 |
| 57 | Revoke (Key) | 키 폐기 | 잃어버린 열쇠 자물쇠 교체 | 노출된 비밀번호를 사용 못 하게 막는 거예요 |
| 58 | Rotate (Key) | 키 교체 | 정기적으로 자물쇠 바꾸기 | 비밀번호를 주기적으로 새것으로 바꾸는 거예요 |
| 59 | Whitelist | 허용 목록 | VIP 명단 | 접근을 허용할 대상만 목록에 적어두는 거예요 |
| 60 | Blacklist | 차단 목록 | 출입금지 명단 | 접근을 차단할 대상을 목록에 적어두는 거예요 |

## 카테고리별 비유 체계

앱 전체에서 일관된 비유를 사용합니다.

| 카테고리 | 비유 세계관 | 예시 |
|---------|-----------|------|
| SECRETS | 가게 비밀번호/금고 | "비밀번호가 쇼윈도에 붙어있어요" |
| AUTH | 가게 출입문/출입증 | "출입문에 잠금이 없어요" |
| DATA | 금고/서류 보관 | "금고 문이 열려있어요" |
| NETWORK | 가게 울타리/담장 | "울타리 없이 뒷문이 열려있어요" |
| INJECTION | 주문서/입력란 | "주문서에 이상한 명령을 끼워넣을 수 있어요" |
| INFRA | 건물 자체 | "건물 기둥에 금이 갔어요" |
| MONITOR | CCTV/감시 카메라 | "CCTV가 꺼져있어요" |
| RESPONSE | 119/소방서 | "소화기가 없어요" |

## 사용 시 주의사항

1. **일관성**: 같은 기술 용어는 항상 같은 일상어로 번역합니다.
2. **병기 가능**: 필요하면 "데이터 열람 권한(RLS)"처럼 일상어를 먼저 쓰고 괄호에 기술용어를 넣을 수 있어요. 하지만 기술용어만 단독으로 쓰면 안 돼요.
3. **비유 확장 가능**: 새로운 비유가 필요하면 이 사전에 먼저 추가하고, 앱 전체에서 일관되게 사용합니다.
