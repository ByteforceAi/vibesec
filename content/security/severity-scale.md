# 심각도 판정 기준 (Severity Scale)

> 각 취약점이 얼마나 급한지 판정하는 기준입니다.
> 건강검진표의 빨강/노랑/초록 표시에 사용됩니다.

---

## 한눈에 보기

| 등급 | 표시 | 뜻 | 한마디 |
|------|------|-----|--------|
| critical | 빨강 | 급해요 | 지금 고치는 게 좋아요 |
| high | 빨강 | 급해요 | 이번 주 안에 고치면 좋겠어요 |
| medium | 노랑 | 조심해요 | 시간 될 때 봐두면 좋아요 |
| low | 초록 | 괜찮아요 | 나중에 해도 돼요 |

---

## 빨강 — 급해요 (Critical / High)

다음 중 **하나라도** 해당하면 빨강입니다.

### 즉시 대응 (Critical)

1. **금전 피해가 바로 생길 수 있어요**
   - 공개된 API 키로 요금 청구 가능 (OpenAI, Anthropic, AWS 등)
   - 결제 시스템 키(Stripe 등) 노출

2. **모든 고객 데이터를 아무나 볼 수 있어요**
   - RLS/Firebase 규칙이 꺼져있어서 전체 DB 공개
   - 인증 없이 다른 사람 데이터 접근 가능

3. **로그인이 의미가 없어요**
   - 인증 완전 우회 가능
   - 관리자 페이지에 아무나 접근 가능

4. **입력란으로 시스템을 조종할 수 있어요**
   - SQL/NoSQL 인젝션으로 데이터 탈취 가능
   - 파일 업로드를 통한 서버 장악 가능

5. **이미 노출/유출이 발생한 정황이 있어요**
   - 비밀번호 파일이 공개 저장소에 있음
   - 이상 요금/이상 접근이 이미 감지됨

### 이번 주 안에 (High)

1. **금전 피해 가능성이 있지만 조건부예요**
   - 비밀번호가 하드코딩돼 있지만 프론트엔드가 아닌 서버에 있음
   - CORS 와일드카드이지만 민감 데이터 API가 아직 없음

2. **부분적 침투가 가능해요**
   - 로그인 시도 무제한 (비밀번호 추측 가능)
   - XSS 저장형 (다른 사용자 세션 탈취 가능)
   - 쿠키 보안 설정 미비

3. **시스템 구조가 외부에 보여요**
   - 에러 메시지에 내부 정보 노출
   - 설정 파일 외부 접근 가능

---

## 노랑 — 조심해요 (Medium)

다음에 해당하면 노랑입니다.

1. **특정 조건 하에 문제가 생길 수 있어요**
   - 프롬프트 인젝션이 가능하지만 민감 정보 접근은 제한적
   - 오픈 리다이렉트 (피싱 연계 가능)
   - 서버 정보 헤더 노출 (직접 피해는 아니지만 공격 도움)

2. **정보 유출 가능하나 금전 피해는 아니에요**
   - 개발 도구가 프로덕션에 켜져있음
   - 디렉토리 리스팅 활성화
   - 에러 메시지에 경로/버전 정보

3. **법적/규정 관련 위험이에요**
   - 개인정보처리방침 없음
   - 탈퇴 사용자 데이터 미삭제
   - 과도한 개인정보 수집

4. **운영 리스크가 있어요**
   - 도메인 만료 위험
   - 환경 미분리 (개발/운영)
   - 테스트/운영 키 혼재

---

## 초록 — 괜찮아요 (Low)

1. **지금 당장은 문제없어요**
   - SSL 만료일 미관리 (아직 유효)
   - 로그 보관 정책 미수립
   - 보안 대시보드 없음

2. **보안 모범 사례 권장 사항이에요**
   - 정기 키 교체 미실시
   - 백업 복구 미테스트
   - 사고 대응 매뉴얼 없음 (아직 사고 없음)

---

## 복합 조건 (여러 취약점이 겹칠 때)

같은 등급이 여러 개 있으면 전체 판정이 올라갈 수 있습니다.

| 조합 | 결과 | 이유 |
|------|------|------|
| critical 1개 이상 | 빨강 (급해요) | 하나라도 위험하면 급해요 |
| high 3개 이상 | 빨강 (급해요) | 약한 곳이 많으면 연쇄 침입이 쉬워요 |
| medium 5개 이상 | 노랑 (조심해요) | 작은 문제도 쌓이면 커져요 |
| 모두 low | 초록 (괜찮아요) | 잘 관리하고 계시네요 |
| high 1-2개 + medium 다수 | 빨강 (급해요) | 전체적으로 약한 상태 |

---

## 점수 산정 방식

건강검진표에 표시되는 종합 점수:

```
총점 = 100 - (critical x 25) - (high x 10) - (medium x 3) - (low x 1)
최솟값: 0점
```

| 점수 | 등급 | 표시 |
|------|------|------|
| 80-100 | 건강해요 | 초록 |
| 50-79 | 조심해요 | 노랑 |
| 0-49 | 급해요 | 빨강 |

---

## 판정 시 주의사항

1. **공포 유발 금지**: 빨강이라고 해서 "큰일 났어요!"라고 말하지 않습니다. "지금 고치는 게 좋아요"라고 말합니다.
2. **맥락 고려**: 아직 사용자가 없는 서비스와 이미 운영 중인 서비스는 급한 정도가 달라요.
3. **실용성 우선**: 고치기 쉬운 것부터 권장합니다. 어려운 것은 패키지로 안내합니다.
4. **과장 금지**: "해킹당했을 수도 있어요"는 확인 전까지 말하지 않습니다.
